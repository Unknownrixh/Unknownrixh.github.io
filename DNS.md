# 🌐 [Day-5] 네트워크 기초: 도메인과 DNS

IP 주소는 컴퓨터가 이해하기엔 좋지만, 사람이 기억하기엔 너무 어렵습니다(예: 142.250.207.110). 그래서 우리는 '도메인'을 사용합니다.

1. 도메인 (Domain)

정의: IP 주소라는 복잡한 숫자 대신 사용하는 사람이 기억하기 쉬운 이름입니다.

예시: google.com, naver.com, github.com

2. DNS (Domain Name System) - "인터넷 전화번호부"

정의: 도메인 이름을 컴퓨터가 이해할 수 있는 IP 주소로 변환해 주는 시스템입니다.

역할: 우리가 웹 브라우저에 google.com을 입력하면, DNS 서버가 "그 주소는 142.250...이야"라고 알려줍니다.

💡 보안 엔지니어의 핵심 포인트! (DNS 보안)

DNS 하이재킹(Hijacking): 해커가 DNS 서버를 속여서, 우리가 접속하려는 사이트 대신 가짜 피싱 사이트로 보내버리는 공격입니다.

보안 엔지니어는 우리가 사용하는 DNS 서버가 안전한지, 도메인 설정이 올바른지 감시해야 합니다.

🛠️ 직접 확인해보기 (실습)

터미널(CMD/Shell)에서 nslookup google.com을 입력해 보세요. 어떤 IP 주소가 나오는지 확인할 수 있습니다!

📝 나의 생각

IP(주소)와 Port(방 번호)를 알게 된 것처럼, 이제 DNS(이름)를 알게 되니 인터넷이라는 거대한 도시의 전체 지도가 머릿속에 그려지기 시작합니다!
