# 🌐 [Day-4] 네트워크 기초: IP와 Port의 관계

클라우드 보안 엔지니어의 핵심 감시 대상인 '포트(Port)'에 대해 정리해 보았습니다.

1. IP (Internet Protocol) - "건물 주소"

정의: 인터넷상에서 데이터를 주고받는 컴퓨터 자체의 고유한 주소입니다.

역할: 데이터를 보낼 '목적지 건물'을 찾는 역할을 합니다.

2. Port (포트) - "방 번호"

정의: 컴퓨터 안에서 실행 중인 특정 프로그램(프로세스)을 구분하기 위한 논리적 문 번호입니다.

역할: 컴퓨터(IP)에 도착한 데이터가 어떤 프로그램(카카오톡, 웹 브라우저, 게임 등)으로 들어가야 할지 안내해 줍니다.

Ex) 친구에게 "빨리 와"라는 문자를 보내기 위해서는 내 휴대폰의 고유한 주소가 IP이고 내가 보낸 "빨리 와"라는 문자가 상대 메세지함으로 데이터를 손실 없이 보내기 위한 규칙이 TCP(하드웨어가 아닌 소프트웨어적 데이터관리), 
    그리고 그 문자가 상대방의 인스타그램이나 카카오톡으로 향하지 않고 메세지함으로 가게 하는 것이 PORT의 역할이다.

💡 보안 엔지니어의 핵심 포인트!

모든 포트(문)를 다 열어두는 것은 위험합니다.

해커는 열려있는 '포트'를 통해 침입합니다.

따라서 우리는 '필요한 포트만 열고 나머지는 닫는(방화벽/Security Group 설정)' 보안 실력을 길러야 합니다.

🛠️ 직접 확인해보기 (실습)

내 컴퓨터의 터미널(CMD/Shell)에서 netstat -an 명령어를 입력해 보면, 현재 어떤 포트들이 열려있는지(LISTENING) 확인할 수 있습니다.
